Maze
- [+] maze must be randomly generated, but reproducibility via a seed is required
- [+] Walls representation: (North, Est, South, West)
- [+] Entry and exit exist and are different, inside the maze bounds
- [+] No isolated cells
- [+] As entry and exist are specific cells, there must be walls at the external borders.
- [+] Each neighbouring cell must have the
same wall if any, e.g., it is forbidden to have the first cell with a wall on the east side, and the second cell behind that wall without a wall on the west side.
- [+] Coridors can't be wider than 2 cells, i.e. not allowed 3x3, it can be 2x3 or 3x2.
- [+] Draw 42 at the center using closed cells
- [+] If there is not space to draw 42, print an error msg
- [+] If the PERFECT flag is activated, the maze must contain exactly one path between
the entry and the exit (i.e., it must be a perfect maze).
- [+] Output File Format (Example: 0011 south:open, west:open, north: closed, east:closed | Cells row by row)
    - 0 (LSB) North
    - 1 East
    - 2 South
    - 3 West
- [+] Cells are stored row by row, one row per line.

- [+] After an empty line
    - Line 1: represent the entrace coordinates
    - Line 2: represent the exit coordinates
    - Line 3: the shortest valid path from
entry to exit, using the four letters N, E, S, W.
    - All lines end with a \n
- [+] Validation of the map by provided validation file
